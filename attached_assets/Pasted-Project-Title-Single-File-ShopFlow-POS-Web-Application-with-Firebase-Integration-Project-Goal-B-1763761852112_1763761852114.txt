Project Title: Single-File "ShopFlow" POS Web Application with Firebase Integration

Project Goal: Build a complete, modern, and professional landing and web application for a Point-of-Sale (POS) system called "ShopFlow." The entire application (HTML, CSS, and JavaScript) must be contained within a single index.html file. No build tools (like Webpack/Vite) or external local CSS/JS files are allowed.

Core Technology Stack:

Structure: Single index.html file.

Styling: Tailwind CSS (via CDN).

Logic: Modern Vanilla JavaScript (ES6+).

Backend: Firebase (Authentication & Firestore) via CDN.

Firebase & Data Architecture (CRITICAL) Initialization:
Use the following global variables for setup (assume these are injected by the host environment): __app_id, __firebase_config, and __initial_auth_token.

Auth Flow: Attempt to sign in using signInWithCustomToken using __initial_auth_token. If undefined, fallback to signInAnonymously.

Standard Auth: Implement UI for Email/Password and Google Sign-In.

Firestore Storage Paths (Strict Adherence Required):

Reviews (Public): Store in /artifacts/{__app_id}/public/data/shopflow_reviews/{documentId}.

User Licenses (Private/Admin): Store in /artifacts/{__app_id}/users/{userId}/shopflow_licenses/{documentId}.

User Profiles: Store in users/{userId} (contains email, role, createdAt).

Real-time Logic:

Use onSnapshot for fetching Reviews and Admin License data to ensure the UI updates instantly without refreshing.

Design & UX Guidelines Aesthetics: Clean, trustworthy SaaS look. Professional Blue (#3b82f6) or Modern Green (#10b981) primary colors. High contrast, plenty of whitespace.
Animations: Implement subtle transitions (fade-ins on scroll, hover effects on buttons/cards) to make the single-file app feel like a premium product.

Responsiveness: Fully responsive design. The Navbar must collapse into a hamburger menu on mobile.

Functional Modules & Sections The application must simulate a Multi-Page Application (SPA) using JavaScript to show/hide sections based on navigation.
